# mtg-followup Skill — 議事録展開 詳細仕様

## 概要

打ち合わせ後の議事録テキストを受け取り、
アクションアイテム・担当者・期限・次回アジェンダを構造化して出力する。
業種横断で利用可能な共通スキル。

## 処理フロー

### Phase 1: 議事録の解析

LLM（Claude）で以下を抽出:

```
抽出項目:
1. 決定事項（"〜することにした" "〜に決定" 等）
2. アクションアイテム
   - タスク内容
   - 担当者（固有名詞 or 役職）
   - 期限（"来週中" → 具体的な日付に変換）
3. 懸案事項・積み残し（"要確認" "次回検討" 等）
4. 次回打ち合わせ候補日（あれば）
```

### Phase 2: 構造化

```markdown
## 決定事項
- [決定内容1]
- [決定内容2]

## アクションアイテム
- [ ] [タスク内容] @[担当者] 期限: YYYY-MM-DD
- [ ] [タスク内容] @[担当者] 期限: YYYY-MM-DD

## 次回アジェンダ案
1. [積み残し議題1]
2. [積み残し議題2]
3. 前回アクションアイテムの進捗確認

## 懸案事項
- [要確認事項1]
```

### Phase 3: Notion 保存（MCP）

Notion MCP で議事録ページを作成:
- データベース: 「議事録」（設定で指定）
- タイトル: `YYYY-MM-DD 打ち合わせ記録`
- プロパティ: 日付、参加者、プロジェクト

**確認ゲートあり**: 保存前に内容を表示して承認を求める。

### Phase 4: 次回アジェンダのSlack送信

Google Calendar MCP で次回打ち合わせを確認し、
アジェンダ案を該当チャンネルに送信。

**確認ゲートあり**: 送信前に内容と送信先を確認。

## 日付変換ルール

| 相対表現 | 変換ルール |
|---|---|
| 来週中 | 翌月曜〜金曜の金曜日 |
| 今月末 | 当月最終営業日 |
| ASAP | 翌営業日 |
| 次回MTGまでに | 次回打ち合わせの前日 |
