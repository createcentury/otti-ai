# freee-check — 未処理明細の自動仕訳

以下の手順で freee の未処理明細をチェックし、仕訳分類してください。

## 引数

$ARGUMENTS（省略可: 事業所IDをスペース区切りで指定。省略時は全事業所）

## 実行手順

1. **データ取得**
   freee MCP を使って、対象事業所の未処理明細を取得してください。
   条件: `未処理 AND 出金 AND 当期会計年度内`

2. **除外ルールの適用**
   CLAUDE.md の「確認ゲート対象」に該当する明細を除外し、
   `logs/YYYY-MM-DD/[company_id]-exceptions.log` に記録してください。

3. **仕訳分類**
   CLAUDE.md の「仕訳分類ルール」に従って分類してください。
   - Phase 1: キーワード辞書マッチング
   - Phase 2: 辞書で判定不能な場合のみLLMフォールバック

4. **重複チェック**
   既存の取引データと「日付・金額・摘要の先頭40文字」を突合し、
   重複がないことを確認してください。

5. **確認ゲート**
   分類結果を表形式で表示し、登録して良いか承認を求めてください。
   承認後に freee MCP で仕訳を登録してください。

6. **レポート生成**
   `reports/YYYY-MM-DD-freee-check.md` に処理サマリを記録してください。
   - 処理件数、人間確認に回した件数、エラー件数
   - 推定所要時間（人間がやった場合）vs 実際の処理時間

7. **CHANGELOG 更新**
   `CHANGELOG.md` に実行結果を1行追記してください。
