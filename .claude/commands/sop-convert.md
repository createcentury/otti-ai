# sop-convert — 業務手順書 → Skill 変換

業務手順書（SOP）のテキストを受け取り、
Claude Code のカスタムスラッシュコマンド（Skill）として利用できるファイルを自動生成します。

これが otti-ai の「自己増殖」機能です。
新しいSOPをSkillに変換することで、フレームワークが継続的に成長します。

## 引数

$ARGUMENTS（SOPのテキストを貼り付け、またはファイルパスを指定）

## 実行手順

1. **SOP の解析**
   入力されたSOPを読み込み、以下を特定してください:
   - この業務の目的（一言で）
   - 入力（何が必要か）
   - 出力（何が生成されるか）
   - 処理ステップ（番号付きで分解）
   - 必要なMCP接続
   - 確認ゲートが必要なステップ

2. **Skill ファイルの生成**
   以下のフォーマットで Skill ファイルを生成してください:
   
   ```markdown
   # [skill-name] — [一言で目的を説明]
   
   [2〜3文で概要を説明]
   
   ## 引数
   $ARGUMENTS（[引数の説明]）
   
   ## 実行手順
   1. [ステップ1]
   2. [ステップ2]
   ...
   ```

3. **ファイル保存**
   - コマンド名を提案（英数字・ハイフンのみ）
   - 保存先を提案: `.claude/commands/[skill-name].md`
   - `skills/[domain]/[skill-name].md` にも詳細版を保存
   - `SKILL.md` のスキル一覧に1行追加

4. **確認**
   生成したSkillファイルの内容を表示し、
   「このコマンド名・内容で保存して良いですか？」と確認してください。

5. **CHANGELOG 更新**
   `CHANGELOG.md` に「新スキル追加: /[skill-name]」を記録してください。
