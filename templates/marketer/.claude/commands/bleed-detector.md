# bleed-detector — 出血検知器（常時監視）

広告アカウントを監視し、費用が出ているのにコンバージョンが発生していない
「出血状態」を検知してアラートを送信します。

## 引数

$ARGUMENTS（省略可: アカウントID。省略時は設定の全アカウント）

## 実行手順

1. **リアルタイムデータ取得**
   Google Ads / Meta Ads MCP に接続し、
   過去24時間の費用・コンバージョン・CPAデータを取得してください。

2. **異常検知ルールの適用**
   CLAUDE.md の「異常検知ルール」に従って以下を確認:
   - 費用超過 × コンバージョンゼロ の広告セット
   - CPA が過去7日平均の2倍を超えたキャンペーン
   - ROAS が目標値の60%を下回ったキャンペーン
   - CTR が0.5%を3日連続で下回っている広告

3. **原因特定（該当があった場合）**
   以下の観点で原因を分析:
   - オーディエンスの疲弊（フリークエンシーの確認）
   - LPの問題（直帰率の急上昇）
   - 入札競合の変化（インプレッションシェアの変化）
   - 季節性・外部要因

4. **アラート送信（確認ゲート不要）**
   異常を検知した場合、Slack MCP で即座にアラートを送信:
   - 対象キャンペーン・広告セット名
   - 異常の内容（数値データ付き）
   - 対象への直接リンク
   - 推定原因

5. **ログ記録**
   `logs/YYYY-MM-DD/bleed-detector.log` に結果を記録してください。
   正常の場合も「正常確認」として記録すること。
