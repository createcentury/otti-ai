# rsa Skill — RSA広告文生成 詳細仕様

## 概要

レスポンシブ検索広告（Responsive Search Ads）用の広告コピーを自動生成する。
見出し15本（各30文字以内）・説明文4本（各90文字以内）を出力し、
Google Ads Editor インポート用CSVとして保存する。

## 処理フロー（詳細）

### Phase 1: コンテキスト取得

```
入力パターンA: キャンペーンID
  → Google Ads MCP で現在の広告・パフォーマンスデータを取得
  → 過去パフォーマンスの上位見出し・説明文を参照

入力パターンB: ブリーフテキスト
  → テキストをそのままコンテキストとして使用
```

### Phase 2: ブランドガイドライン確認

CLAUDE.md の「ブランドボイス・ガイドライン」を参照:
- トーン・スタイル
- 使用禁止ワード
- 文字数制限（見出し30文字、説明文90文字）
- 必須含有要素（ブランド名、CTA等）

### Phase 3: サブエージェント分業生成

#### 見出しサブエージェント（30文字制限厳守）

生成カテゴリ（各3本以上）:
- 製品ベネフィット訴求
- 機能・スペック訴求  
- CTA（行動喚起）
- 社会的証明・実績
- 緊急性・限定性

ピン留め候補（位置固定）:
- Headline 1: ブランド名 + 主要ベネフィット（3候補）

#### 説明文サブエージェント（90文字制限厳守）

生成カテゴリ（各1〜2本）:
- 詳細ベネフィット説明
- 社会的証明（数字・実績）
- 安心感・信頼訴求
- CTA + 限定オファー

### Phase 4: 自己検証ループ

```python
def validate_rsa(headlines, descriptions):
    errors = []
    for h in headlines:
        if len(h) > 30:
            errors.append(f"見出しオーバー: {h} ({len(h)}文字)")
    for d in descriptions:
        if len(d) > 90:
            errors.append(f"説明文オーバー: {d} ({len(d)}文字)")
    # 禁止ワードチェック
    # ブランドボイス確認
    return errors
```

エラーがある場合: 自動修正して再検証（最大3回）
3回失敗: 人間に確認を求める

### Phase 5: 出力

#### CSV出力（Google Ads Editor インポート用）

```csv
Campaign,Ad Group,Headline 1,Headline 2,...,Description 1,...
```

保存先: `reports/YYYY-MM-DD-rsa-[campaign_id].csv`

#### テーブル表示（人間レビュー用）

```
見出し一覧:
  1. [30文字以内]  ✓
  2. [30文字以内]  ✓
  ...

説明文一覧:
  1. [90文字以内]  ✓
  ...
```

### Phase 6: 確認ゲート

広告の公開・適用前に承認を求める。
承認後: Google Ads MCP で広告を更新。

## パフォーマンス最適化のヒント

- 過去パフォーマンスの上位見出しはそのまま再利用する
- 新しいバリエーションを3〜5本追加してA/Bテスト
- 品質スコアが低い広告の見出しパターンは避ける
